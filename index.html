Quickstart

There are 2 main ways to use the Citadel tools. We recommend to start by using the tools on the Citadel plateform, which doesn't require any technical skill, and then switch to the Github tools if you want more control on the resulting apps.

Step 1 : Get and prepare your data

Prepare a spreadsheet data file for the converter (see pre-requisites, see sample spreasheet)

Step 2 : Convert your data to Citadel JSON

Use the Citadel table to JSON Converter tool to convert your CSV or XLS files into Citadel JSON that will be directly usable into your apps (see Converter tutorials, FAQ and How-tos)

Step 3 : Make your app !

EASIEST : Use the Citadel platform

  1. Publish your converted JSON files to the Citadel datastore
  2. Use the Application Generator to make it an app ! (see AGT tutorial)

EASY : Use the Citadel mobile applications templates on Github

(see detailed Templates tutorials)

  1. Choose and fork a template - or download it to your computer (see Templates differences and uses below)
  3. Add your converted data into the template
  5. Play with the settings and tweak the template to fit your needs !

How to use the Github repositories

  • The Data converter : converts CSV or Excel files to Citadel JSON files that are ready to be used into application templates
    • Converter library
    • Converter GUI
    • Converter portlet (Liferay)
  • The Application Generator Tool (AGT) : this tools makes it easy to build an app without any technical skill. It is the source code of the tool that is available on Citadel platform.
  • The mobile application templates : these templates are meant to be forked and used to quickly deploy mobile applications. They are build on HTML5 and use the Citadel JSON file format. Most templates will work "out-ofthe-box" with none or very few changes. You are encouraged to build application upon them, and of course contribute back your improvements so they can benefit other people. The various templates demonstrate various uses of the mobile applications, and implement several functionnalities. Most templates use static JSON files, but can be set to use live data and/or a database :
    • POIs template : use this template to display POI (Points Of Interest = places) on a map. Different categories can be used so the user can switch between the different categories that are displayed on the map.
    • Parkings template : use this template to display live parking data, including their location, and the number of parkings left.
    • Events template : use this template to display events. This template provides a calendar filter so events can be filtered by proximity and by date and time.
    • Environnemental template : use this template if you wish to display live sensor data such as temperature, live air analysis, etc. [currently uses Xively for the sensor interface]
    • Crowd-sourcing template : use this template if you want thet app users to add new data to your dataset. [requires a database]

Contribute !

Citadel is looking for your feedbacks, use case, stories, and to hear about anything that you have made with our tools ! Please use the following channels to get in touch with us:

  • The Citadel Community is hosted on the Citadel on the Move website. It's a public ressource, but you'll need to create a account to contribute.
    • Use the forum to discuss about the AGT, applicaiton or data templates, or anything related to the Citadel project itself
    • Ask a question and vote for relevant answers
    • Ask for help and help us improve the FAQ
  • Feedbacks and technical issues can be submitted through the Github Issues tool : select the relevant repository, and submit your issue / enhancement / feature request.
  • We love to hear from your experiences and stories with Citadel : feel free to send us your links to blog post, mobile applications or other stuff you've made with Citadel through one or more of the following channels :
    • Share a link with our Twitter account @citadel_eu
    • Post on our Facebook page
    • Add an issue here on Github so we can add your links into our Tutorials / Stories / Applications section
